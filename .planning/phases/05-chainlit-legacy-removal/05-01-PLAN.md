# Plan 05-01: Chainlit Legacy Removal (single consolidated plan)

<objective>
Chainlit 코드/의존성/설정/문서 전부 제거. Slack bot 경로(`main.py`, `bot/`, `law/`)는 diff 0. Phase 5는 독립 플랜이라 다른 phase 아티팩트와 충돌 없음.

단일 commit (삭제 묶음) + 1 commit (문서 업데이트/SUMMARY) + 1 commit (Slack bot smoke test result) 지향. 3 commit max.
</objective>

<task id="5-01">
<title>Chainlit 파일/디렉터리 일괄 삭제 + 설정 파일 편집</title>

<read_first>
- app.py (삭제 전 내용 확인, 정말 Chainlit 루트 진입점인지)
- chainlit.md (존재 시 삭제 전 내용 확인)
- .chainlit/ (ls로 구조 확인)
- frontend/src/app/test-sidebar/ (ls로 파일 목록 확인)
- requirements.txt (chainlit 라인 위치 확인)
- Dockerfile (Chainlit 참조 범위 확인 → 최소 편집 vs 재작성 결정)
- .env.example (CHAINLIT_* 변수 목록 확인)
- main.py (보존 대상 — 삭제 전 diff 기준)
</read_first>

<action>

### 1. 파일 삭제

```bash
rm -f app.py chainlit.md
rm -rf .chainlit/
rm -rf frontend/src/app/test-sidebar/
```

각 삭제 전 `ls -la` 로 존재 확인, 존재하지 않으면 skip. `git rm` 은 autoremove 되므로 일반 `rm` 사용 후 `git add -A` 로 스테이지.

### 2. requirements.txt 편집

`chainlit` 라인만 제거. 다른 라인 (`slack-bolt`, `google-generativeai`, `httpx`, `python-dotenv` 등) 그대로.

### 3. Dockerfile 편집

먼저 파일 내용 확인. 다음 경로 중 선택:
- **(a) Chainlit 중심 구조면 재작성**: base image (python:3.11-slim 등) → COPY + pip install -r requirements.txt → CMD ["python", "main.py"]. EXPOSE 삭제.
- **(b) Chainlit 라인만 몇 개면 최소 편집**: chainlit 관련 ENV/CMD/EXPOSE 7860 라인만 제거, 나머지 보존.

Slack bot은 outbound Slack API를 쓰므로 EXPOSE 포트 불필요.

### 4. .env.example 편집

`CHAINLIT_AUTH_SECRET`, `CHAINLIT_AUTH_TOKEN`, `CHAINLIT_*` 패턴 라인 전부 제거. `SLACK_BOT_TOKEN`, `SLACK_APP_TOKEN`, `LAW_API_KEY`, `GOOGLE_GENERATIVE_AI_API_KEY`, 기타 모두 보존.

### 5. grep 검증

```bash
grep -ri "chainlit" . \
  --exclude-dir=.git \
  --exclude-dir=node_modules \
  --exclude-dir=.next \
  --exclude-dir=.planning \
  --exclude-dir=.claude \
  2>&1
```

결과가 0건이어야 함. `.planning/` 안에는 Phase 5 문서 자체가 "chainlit"을 포함하므로 제외.

frontend/.next 디렉터리는 build artifact이고 이전 dev 세션의 캐시가 chainlit 레퍼런스를 포함할 수 있음 — build artifact라 제외.

### 6. Slack bot smoke test (선택, orchestrator 재량)

Plan CONTEXT D-07은 `python main.py` 수동 기동을 요구하지만, orchestrator는 Python 서브프로세스를 대화형으로 띄우기 어려움. 대안:

- `python -c "import main; print('main.py import OK')"` — import만 성공해도 syntax/의존성 회귀 없음
- 또는 `python main.py` 를 background로 띄우고 3-5초 후 로그 grep으로 "Slack Bolt" 또는 유사 startup 메시지 확인

둘 다 실패하면 SUMMARY에 "smoke test skipped — Python 환경 미가동"으로 기록하고 carry-forward.

### 7. 단일 commit

```bash
git add -A
git commit -m "chore(05): remove Chainlit legacy (CLEAN-01..06)"
```

또는 더 세분화 (파일 삭제 → 설정 편집 2단계)도 허용. 총 1-2 commit.
</action>
